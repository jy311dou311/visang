# 가치 말하기 구현

## **📌** 프로젝트 개요

사내 핵심 가치 ‘우리의 믿음’을 주제로 한 팝업형 참여 프로젝트의 일환으로 진행된 팀 프로젝트입니다. 
본 프로젝트는 여러 개의 참여형 콘텐츠로 구성된 팝업 프로젝트이며, 그중 저는 지도 기반 ‘가치 말하기’ 모듈의 기획과 구현을 담당하였습니다. 

## **🧠** 문제 정의

명확한 핵심 가치가 있는 문장이지만 이를 구성원 개인의 경험과 연결되기에는 다소 추상적이라는 한계가 있었습니다. 또한 이러한 공지나 캠페인의 방식은 일방적인 전달에 그치고 참여 부담이 높으며 기억에 오래 남지 않는 문제가 발생할 수 있습니다.
따라서 저희는 “가치를 설명하지 않고 구성원이 직접 떠올리고 말하게 만들 수 없을까?” 하는 문제의식에서 프로젝트를 시작하였습니다. 

## ⚙️ 사용 기술

-HTML

## **🔍 구현 방식**

1. 핵심 가치를 공간 개념으로 재해석하여 지도 형태의 UI로 구성
2. 각 구역마다 가치 문장을 직접 묻지 않고, 일상 경험을 떠올릴 수 있는 질문으로 변환
3. 사용자가 한 줄 경험을 입력하면 데이터베이스에 저장
4. 수집된 응답을 이후 부서별로 정리, 공유할 수 있도록 구조 설계

팀 내 다른 구성원들은 
- 다른 참여 콘텐츠 기획
- 시각 디자인
- 전체 팝업 운영 및 동선 설계를 담당했습니다. 

## ⁉️ 문제점&해결과정

초기 기획 단계에서는
- 질문이 여전히 추상적으로 느껴져 참여를 망설일 수 있다는 의견이 있었고
- 이를 구현하는 것에 한계가 있었습니다.

이를 해결하기 위해
- 질문 문장을 일상적인 상황을 바로 떠올릴 수 있는 형태로 재구성하고 
- Firebase 등을 이용하여 간단하게 구현하였습니다.

이 과정을 통해 작은 문장의 차이가 참여율에 직접적인 영향을 미치다는 점과 기획과 구현이 분리되지 않는다는 점을 체감할 수 있었습니다. 

## 🤔 결과 및 한계

팀원들끼리 참여해 본 결과 간단한 질문으로 부담 없이 참여하며 각자의 표현으로 핵심 가치를 해석한 기록을 축적할 수 있었습니다. 지도 기반 시스템 덕분에 참여 자체를 하나의 가벼운 탐색 경험으로 인식하게 만들 수 있었습니다.

다만 정성 데이터 중심의 프로젝트이기에 결과를 수치로 비교하거나 자동 분석하기에는 한계가 있었고, 단발성 이벤트 이후의 지속성은 추가적인 설계가 필요하다는 것을 확인하였습니다. 

## 🙋‍♀️ 담당한 역할

- 참여 흐름 설계 
- 질문 문장 기획 및 가치 재해석
- 웹 화면 구현 및 Firebase 연동
